
# Playwright TypeScript Test Framework 

## 📝 Опис проєкту
Це базова структура проєкту для автоматизованого тестування вебзастосунків з використанням Playwright та TypeScript. Проєкт підготовлений як надійний фундамент для подальшої розробки автотестів із застосуванням патернів Page Object Model (POM) та Component-based архітектури.


## 1. Клонування репозиторію
```bash
git clone https://github.com/Vaz1k28/TAQC-Project-Team_4.git
cd TAQC-Project-Team_4 
```

## 2. Встановлення залежностей
Переконайтеся, що у вас встановлено Node.js
```bash
npm install
```
## 3.Встановлення браузерів Playwright
Playwright потребує власних бінарних файлів браузерів для коректної роботи
```
npx playwright install
```

## ⚙️ Конфігурація
Проєкт використовує змінні оточення для гнучкості налаштувань.

   1.Створіть файл .env у корені проєкту.

   2.Скопіюйте вміст із .env.example та налаштуйте під свої потреби:
   
```
BASE_URL=https://example.com
HEADLESS=true
RETRIES=0
TIMEOUT=30000
```

## Запуск тестів
Ви можете використовувати стандартні команди Playwright 
```
npx playwright test 
```
або скороченим скриптом
``` 
npx test
``` 

## Запуск тестів у UI режимі
```
npx playwright test --ui
```

## 📁 Структура проєкту
| Директорія / | Призначення |
| :--- | :--- |
| **`tests/`** | Файли тестів (`*.spec.ts`). Містять лише логіку перевірок. |
| **`pages/`** | Опис сторінок застосунку. Містять локатори та методи взаємодії. |
| **`components/`** | Повторювані елементи (Header, Footer, Modals), що використовуються на різних сторінках. |
| **`fixtures/`** | Розширення стандартного об'єкта `test` (напр., автоматична авторизація). |
| **`utils/`** | Допоміжні функції: робота з API, логування, обробка `ENV`. |
| **`playwright.config.ts`** | Головний конфігураційний файл фреймворку. |

## 🛠 Ключові компоненти архітектури

*   **`pages/base.page.ts`** — Абстрактний клас, від якого успадковуються всі сторінки. Забезпечує доступ до базових методів Playwright.
*   **`components/base.component.ts`** — База для UI-фрагментів, що дозволяє інкапсулювати логіку компонентів окремо від сторінок.
*   **`utils/env.ts`** — Модуль для безпечного зчитування та валідації змінних середовища.
